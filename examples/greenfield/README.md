# Example — Greenfield Project

> Empty folder, fresh start, full pipeline.

## Scenario

You're starting a new project. You want the full Eidolons team from day one — scout, planner, coder, chronicler — with reasoner on the sidelines.

## Setup

```bash
mkdir new-project
cd new-project
git init
eidolons init --preset pipeline
```

## `eidolons.yaml`

```yaml
# eidolons.yaml — per-project manifest
# Generated by eidolons v1.0.0 at 2026-04-21T14:30:00Z
# Docs: https://github.com/Rynaro/eidolons/blob/main/docs/getting-started.md

version: 1
hosts:
  wire: [claude-code, copilot, cursor, opencode]

members:
  - name: atlas
    version: "^1.0.0"
    source: github:Rynaro/atlas
  - name: spectra
    version: "^4.2.0"
    source: github:Rynaro/SPECTRA
  - name: apivr
    version: "^3.0.0"
    source: github:Rynaro/apivr
  - name: idg
    version: "^1.1.0"
    source: github:Rynaro/idg

composition:
  pipeline: [atlas, spectra, apivr, idg]
```

## First use

In Claude Code or Copilot:

```
@atlas mission — map the proposed architecture from this brief:
"A Rails API + Next.js frontend with a PostgreSQL backend.
Feature flags via Flipper. Background jobs via Sidekiq."

Decision target: entity list + module boundaries + key decisions needed before implementation.
```

ATLAS responds with a scout-report. You hand it to SPECTRA:

```
@spectra — here's the ATLAS scout-report. Produce a spec for "Phase 1:
authentication + user profile CRUD" with all validation gates.
```

SPECTRA produces a spec. You hand it to APIVR-Δ for implementation. Then IDG chronicles the session.
