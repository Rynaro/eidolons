## Intent

Rewrite `README.md` for the v2.0→v2.3 era. Not an incremental edit: a research-grounded
recomposition. Requested by the maintainer with four explicit goals:

1. **Grounded in evidence** — patterns mined from celebrated OSS READMEs and from
   peer-reviewed attention/reading psychology (research reports commissioned for this change).
2. **Dual audience** — beginners must understand; experts must get curious.
3. **Persona with credibility** — the Eidolons voice (named specialists, the summon motif)
   without losing the measured-not-marketed evidence posture.
4. **Model-tier-conscious production** — haiku for mechanical inventory, sonnet for research
   breadth, opus for judgment-heavy critique/verification, Fable for synthesis and composition.

## Scope

- `README.md` only. No roster, CLI, schema, or methodology changes.
- Dynamic shields.io release badges must remain dynamic (see the in-file DYNAMIC comment).
- All measured numbers (compliance A/B, Vivi fanout, Kupo additive-proof, H-WIN, RAMZA AC-003)
  keep their honest caveats — floor/ceiling framing is a feature, not noise.

## Acceptance checks

- **AC-RM-1** — every relative link resolves to an existing file/dir (mechanical).
- **AC-RM-2** — every quantitative claim traceable to CHANGELOG.md / roster/index.yaml /
  evals/ / .spectra/research/ (mechanical cross-check).
- **AC-RM-3** — independent cold-read by the distinct checker: a beginner can state what the
  project is and the first command to type within 60 seconds; an expert names at least one
  hook that made them want to click deeper.
